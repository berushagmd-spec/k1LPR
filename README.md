# Kurbani Alert Bot

Telegram-бот на Python для атмосферного автопостинга сводок по сеттингу Курбании/Турбании.

Проект сделан под канал `-1004449922425` и админа `7787565361`.  
Бот генерирует ветвящиеся события: опасность, тревога, повторно, расширение зоны, сохранение угрозы и отбой. БПЛА сделаны редкими, основа генерации — авиация, ракеты и артиллерия. Внутри используются только названия из сеттинга Курбании/Турбании.

Важно: используйте бота только в канале, где аудитории понятно, что это лор/ролеплей/игровой проект. Генератор не использует реальные населённые пункты из примеров ЛПР.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# .venv\Scripts\activate    # Windows

pip install -r requirements.txt
cp .env.example .env
```

Откройте `.env` и вставьте токен бота от BotFather:

```env
BOT_TOKEN=123456:ABCDEF...
```

Проверьте, что бот добавлен администратором в канал и имеет право публиковать сообщения.

Запуск:

```bash
python bot.py
```

## Команды админа

Команды работают только для ID, указанных в `ADMIN_IDS`.

`/start` или `/help` — показать помощь.  
`/status` — текущее состояние, канал, режим, интервалы, активные события.  
`/preview` — сгенерировать пример без публикации.  
`/sample 5` — показать несколько примеров без публикации.  
`/post_now` — сгенерировать и сразу отправить пост в канал.  
`/custom текст` — отправить свой текст в канал.  
`/pause` — остановить автопостинг.  
`/resume` — включить автопостинг.  
`/set_interval 15 45` — изменить интервал автопостинга.  
`/set_channel -1004449922425` — изменить канал публикации.  
`/mode calm|normal|hot|chaos` — интенсивность событий.  
`/scenario mixed|bpla|missile|aviation|artillery` — профиль событий.  
`/active` — показать активные ветки тревог.  
`/clear_active` — очистить активные ветки без публикации отбоя.

## Режимы

`calm` — редкие события, чаще отбой и короткие предупреждения.  
`normal` — сбалансированный режим, БПЛА появляются редко.  
`hot` — больше повторов, расширений зон и комбинированных угроз, но БПЛА все равно не основной тип.  
`chaos` — самый плотный режим, много ветвлений и повторных сообщений.

## Как устроена логика

Бот хранит активные тревоги в `state.json`.  
Когда автопостинг срабатывает, генератор решает:

1. создать новую ветку;
2. повторить существующую;
3. расширить угрозу на соседние города;
4. усилить тип угрозы;
5. дать отбой.

Фронт не двигается. Слова вроде “прифронт”, “тыл”, “ближний тыл” не публикуются — они используются только внутри логики выбора событий.

## Автозапуск через systemd

Создайте `/etc/systemd/system/kurbani-alert-bot.service`:

```ini
[Unit]
Description=Kurbani Alert Telegram Bot
After=network.target

[Service]
WorkingDirectory=/opt/kurbani_alert_bot
ExecStart=/opt/kurbani_alert_bot/.venv/bin/python /opt/kurbani_alert_bot/bot.py
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

Затем:

```bash
sudo systemctl daemon-reload
sudo systemctl enable kurbani-alert-bot
sudo systemctl start kurbani-alert-bot
sudo systemctl status kurbani-alert-bot
```
